# Что нужно сделать ТЕБЕ (пошагово)

Это твой личный чеклист. Claude Code напишет код, но аккаунты, оплаты, голос и аватар — твоя часть. Порядок соответствует этапам ТЗ: для Этапа 1 нужны только шаги 1–3.

---

## ШАГ 1. Telegram-бот (5 минут, бесплатно)

1. Открой в Telegram бота **@BotFather** → команда `/newbot`.
2. Придумай имя (например «Ильфат | Контент-эксперт») и username (например `ilfat_avatar_bot`).
3. Скопируй **токен** → это `TELEGRAM_BOT_TOKEN`.
4. Узнай свой числовой Telegram ID: напиши боту **@userinfobot** → это `ADMIN_TELEGRAM_ID`.
5. Опционально у BotFather: `/setuserpic` (твоё фото), `/setdescription` (описание).

## ШАГ 2. Anthropic API (10 минут)

1. Зайди на **console.anthropic.com** → зарегистрируйся.
2. Пополни баланс (нужна банковская карта; минимум $5 хватит надолго для теста).
3. Settings → API Keys → **Create Key** → это `ANTHROPIC_API_KEY`.

Ориентир по стоимости: с промпт-кэшированием один текстовый ответ бота обойдётся в доли цента. $5–10/мес для личного бота — реалистичный бюджет.

## ШАГ 3. Подготовить persona.md и базу знаний

Без этого бот будет «обычным ChatGPT», а не тобой. Подробно — в разделе «База знаний» ниже. Минимум для старта: заполнить шаблон `persona.md` (Claude Code его создаст) и положить 2–3 файла с твоими материалами в папку `knowledge/`.

✅ **После шагов 1–3 можно запускать Этап 1.**

---

## ШАГ 4. ElevenLabs — твой голос (30–60 минут)

1. Зарегистрируйся на **elevenlabs.io**.
2. Тариф: для старта хватит **Starter ($5/мес)** — даёт Instant Voice Clone (клон по 1–2 минутам записи) и ~30 минут озвучки в месяц. Если хочешь максимальное сходство голоса — **Creator ($22/мес)** с Professional Voice Clone (нужно 30+ минут чистых записей, обработка несколько часов). Совет: начни со Starter, апгрейднешься если качество не устроит.
3. **Запиши образец голоса**: 2–3 минуты обычной речи. Требования: тихая комната, говори как обычно говоришь в кружочках (не читай монотонно!), без музыки и шумов, телефонного диктофона достаточно, но лучше микрофон/гарнитуру.
4. В ElevenLabs: **Voices → Add voice → Instant Voice Clone** → загрузи запись → дай имя.
5. Открой созданный голос → скопируй **Voice ID** → это `ELEVENLABS_VOICE_ID`.
6. Профиль (иконка справа внизу) → **API Keys** → создай ключ → это `ELEVENLABS_API_KEY`.
7. Проверь сразу на сайте: вбей русский текст, послушай. Если звучит плохо — перезапиши образец качественнее.

## ШАГ 5. HeyGen — твой видео-аватар (1–2 часа + ожидание обработки)

### Запись видео для аватара — самое важное

От качества этой записи зависит, насколько живыми будут кружочки. Рекомендации:

- **2–5 минут видео**, где ты говоришь в камеру.
- Камера на уровне глаз, **iPhone на штативе — отлично** (снимай в 4K или 1080p).
- Ровный мягкий свет на лицо (окно перед тобой или кольцевая лампа), **нейтральный однотонный фон**.
- Кадрирование: грудь и голова в кадре, вокруг головы запас (для квадратной обрезки под кружочек).
- Поведение: смотри в объектив, говори спокойно, естественная мимика, лёгкие кивки. **Руки не подноси к лицу**, не раскачивайся, не уходи из кадра. Между фразами делай паузы с закрытым ртом и нейтральным лицом — из таких пауз система берёт «слушающее» состояние.
- Одежда без мелких полосок (рябит).

### Настройка

1. Зарегистрируйся на **heygen.com** → создай аватар: раздел **Avatars → Create Avatar → Video Avatar** → загрузи запись → подтверди согласие (запишешь короткую фразу-разрешение) → дождись обработки (минуты–часы).
2. Проверь аватар в веб-интерфейсе: сгенерируй тестовое видео с русским текстом, оцени липсинк.
3. **API-доступ**: у HeyGen API оплачивается отдельно от веб-подписки — зайди в Settings → API (или heygen.com/api), посмотри актуальные тарифы. Для старта обычно есть бесплатные кредиты на тесты. Скопируй **API Key** → это `HEYGEN_API_KEY`.
4. ID аватара: в интерфейсе аватара или через API-эндпоинт списка аватаров (Claude Code поможет вытащить) → это `HEYGEN_AVATAR_ID`.

💰 Важно понимать: кружочки — самая дорогая часть (порядка $0.3–1 за штуку через API в зависимости от тарифа). Поэтому в ТЗ заложен дневной лимит кружочков на пользователя и кэш приветственного кружочка.

## ШАГ 6. Где это будет работать

- **Для разработки и тестов**: прямо на твоём MacBook (бот работает, пока запущен терминал). Понадобится установленный `ffmpeg` (`brew install ffmpeg`).
- **Для постоянной работы**: любой дешёвый VPS (1–2 ГБ RAM, Ubuntu, ~300–500 ₽/мес). Claude Code напишет инструкцию по деплою, когда Этап 1 будет готов — просто попроси.

# База знаний: куда грузить и как обновлять

## Как это устроено

База знаний — это просто **папка `knowledge/` рядом с ботом**, в ней обычные `.md`-файлы. Никаких векторных баз и сторонних хранилищ. Бот при каждом ответе видит всё содержимое папки целиком (благодаря кэшированию это почти бесплатно).

## Самый удобный способ обновлять — через сам Telegram 📲

В ТЗ заложено: ты (как админ) **просто отправляешь боту файл `.md` или `.txt` в личку** — он сохраняет его в базу знаний и сразу начинает использовать. С телефона, из любой точки, без серверов и консолей.

Рабочий процесс выглядит так:

1. Пишешь/обновляешь материал где удобно (Google Docs, Notion, Apple Notes, или прямо в чате со мной — «Клод, оформи мои мысли про контент-воронки в md-файл»).
2. Сохраняешь как `.md` или `.txt`.
3. Кидаешь файл боту → готово. Файл с тем же именем перезаписывается — так и обновляешь темы.

Дополнительно: `/kb` — посмотреть что в базе, `/kb_delete` — удалить файл.

## Как структурировать материалы (рекомендации)

- **Один файл = одна тема.** Например: `контент-стратегия.md`, `reels-сценарии.md`, `воронки-продаж.md`, `мои-кейсы.md`, `прайс-и-услуги.md`, `faq.md`.
- Внутри файла — заголовки `##`, короткие абзацы, списки. Чем чище структура, тем точнее бот отвечает.
- Пиши **своими формулировками и со своими примерами** — именно это отличает «твоего» бота от обычного ИИ.
- Обязательно сделай файл `faq.md`: 15–20 реальных вопросов, которые тебе задают, с твоими ответами — это сильнее всего влияет на качество.
- Ориентир по объёму: суммарно до ~150–200 страниц влезает без проблем. Ты вряд ли упрёшься в лимит; если упрёшься — в ТЗ заложен автоматический переход на режим подгрузки файлов по запросу.

## Откуда брать контент для базы

- Экспорт из Google Docs / Notion (оба умеют экспорт в Markdown, Google Docs — «Файл → Скачать → Markdown»).
- Расшифровки твоих видео/Reels/выступлений (можно прогнать через тот же ElevenLabs Scribe или попросить меня структурировать сырую расшифровку).
- Твои старые посты, гайды, чек-листы, презентации — скидывай мне, я конвертирую в чистые md-файлы для базы.
